# time-wheel-scheduler
## 基于 C++17 高性能分层时间轮调度器
## 项目简介
本项目基于 C++17 标准实现轻量化分层时间轮调度器，是后端服务领域经典的定时与延时任务解决方案。
相较于最小堆、红黑树等传统定时器实现方式，时间轮算法拥有稳定 O(1) 的任务插入、删除与触发效率，规避频繁排序带来的性能损耗。
适用于高并发后端服务、游戏网关、连接超时管理、订单延时关闭、异步任务回调等业务场景。

项目采用纯标准库开发，无任何第三方依赖，遵循模块化工程设计规范。通过独立调度线程实现任务异步轮转，具备完善的多线程安全保障，支持跨平台编译部署，可快速集成至各类 C++ 后端工程。

## 核心特性
1. 采用秒、分、时三级分层时间轮架构，任务调度时间复杂度稳定为 O(1)
2. 独立后台调度线程驱动轮转，不阻塞业务主线程执行逻辑
3. 基于互斥锁与原子变量实现线程安全，支持多线程并发新增、取消延时任务
4. 纯 C++17 标准库实现，轻量化架构，无第三方库冗余依赖
5. 模块化接口设计，代码结构清晰，便于业务扩展与二次开发
6. 适配 CMake 工程构建，支持 Linux、WSL、Windows、macOS 跨平台编译
7. 内置完整调用示例，可直接编译运行，快速上手开发调试

## 架构设计
### 分层时间轮模型
整体采用三级时间轮逐级进位设计：
秒级时间轮，1秒时间粒度，划分60个刻度，负责短时延时任务管理；
分钟级时间轮，1分钟时间粒度，划分60个刻度，秒轮走完一圈后任务自动进位；
小时级时间轮，1小时时间粒度，划分24个刻度，分钟轮走完一圈后任务自动进位。

任务跟随时间轮刻度自动轮转，到期即刻触发回调函数，无需全局遍历检索，最大限度降低CPU资源消耗。

### 线程模型
业务线程：处理业务请求，负责动态提交延时任务、撤销已有任务；
调度线程：独立后台常驻线程，专职完成时间轮刻度步进、到期任务检测、回调函数执行；
同步机制：对时间轮节点、任务队列做严格线程同步保护，保障高并发场景下的数据安全。

## 基础使用示例

// 启动后台调度线程
scheduler.start();
